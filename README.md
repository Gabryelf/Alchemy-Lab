## Общее описание проекта

Это простая 2D игра, созданная для изучения объектно-ориентированного программирования (ООП) и библиотеки Pygame. Игрок управляет зеленым квадратом, который может стрелять желтыми пулями по красным врагам. Враги появляются с краев экрана и движутся к игроку.

**Основные механики:**
- Управление персонажем с помощью стрелок
- Стрельба по клику мыши
- Враги имеют 2 очка здоровья
- Подсчет очков за убийство врагов
- При столкновении с врагом игра возвращается в меню

## Класс Game

**Назначение:** Главный класс, управляющий всем игровым процессом.

### Метод `__init__(self)`

**Назначение:** Инициализация игры, создание всех объектов и настройка начального состояния.

**Что делает:**
- Запускает Pygame и создает окно
- Устанавливает игровой таймер
- Создает игрока, оружие и пустой список врагов
- Инициализирует UI элементы (панель, меню)
- Устанавливает начальное состояние игры (меню)

**Комментарий:** Этот метод вызывается автоматически при создании объекта Game. Здесь происходит вся начальная настройка: от создания окна до подготовки игровых объектов.

---

### Метод `handle_events(self)`

**Назначение:** Обработка всех событий ввода (клавиатура, мышь, закрытие окна).

**Что делает:**
- Проверяет, не хочет ли игрок закрыть окно
- При клике мыши в игре создает новую пулю
- Обрабатывает нажатия на кнопки UI
- Переключает состояние игры (меню/игра)

**Комментарий:** Этот метод запускается каждый кадр и проверяет, что сделал игрок. Если игрок кликнул мышью во время игры, создается новая пуля. Если нажата кнопка "MENU" - происходит возврат в меню.

---

### Метод `reset_game(self)`

**Назначение:** Сброс игры до начального состояния при новом старте.

**Что делает:**
- Создает нового игрока в центре экрана
- Создает новое оружие
- Очищает список врагов
- Обнуляет счет и таймеры

**Комментарий:** Этот метод вызывается, когда игрок нажимает "START" в меню. Он полностью очищает игру и начинает ее заново, чтобы предыдущие враги и очки не мешали.

---

### Метод `spawn_enemy(self)`

**Назначение:** Создание нового врага на одном из четырех краев экрана.

**Что делает:**
- Выбирает случайную сторону экрана (лево, право, верх, низ)
- Устанавливает координаты врага за пределами экрана
- Создает объект Enemy с этими координатами
- Добавляет врага в список enemies

**Комментарий:** Враги всегда появляются за пределами экрана и затем идут к игроку. Это создает эффект "нападения со всех сторон". Сторона выбирается случайно, чтобы игра была непредсказуемой.

---

### Метод `check_collisions(self)`

**Назначение:** Проверка столкновений между пулями и врагами.

**Что делает:**
- Проходит по всем пулям и врагам
- Проверяет, пересекаются ли их прямоугольники
- При попадании удаляет пулю и наносит урон врагу
- Если враг умер (hp <= 0), удаляет его и добавляет очко

**Комментарий:** Это сердце боевой системы. Метод использует простую проверку прямоугольников (colliderect) для определения попаданий. Важно проходить по копии списков, чтобы безопасно удалять элементы во время итерации.

---

### Метод `draw(self)`

**Назначение:** Отрисовка всех объектов на экране.

**Что делает:**
- В режиме меню рисует меню
- В режиме игры:
  - Очищает экран черным цветом
  - Рисует игрока
  - Рисует всех врагов
  - Рисует все пули
  - Показывает счет и количество пуль
- Рисует верхнюю панель с кнопкой меню

**Комментарий:** Отрисовка происходит каждый кадр. Сначала заливается фон, затем поверх рисуются все игровые объекты. В самом конце вызывается pygame.display.flip() для обновления экрана.

---

### Метод `update(self)`

**Назначение:** Обновление логики игры (движение объектов, спавн врагов).

**Что делает:**
- Двигает игрока по нажатым клавишам
- Обновляет позицию оружия (следует за игроком)
- Обновляет все пули (движение и удаление улетевших)
- Двигает всех врагов к игроку
- Создает новых врагов каждые 60 кадров
- Проверяет коллизии и столкновение игрока с врагом

**Комментарий:** Это самый важный метод для игровой логики. Он вызывается каждый кадр и обновляет позиции всех движущихся объектов. Таймер enemy_spawn_timer считает кадры и создает нового врага, когда достигает 60 (1 секунда при 60 FPS).

---

### Метод `run(self)`

**Назначение:** Главный игровой цикл.

**Что делает:**
- Запускает бесконечный цикл while
- В каждой итерации:
  - Обрабатывает события
  - Обновляет логику
  - Рисует кадр
  - Ограничивает FPS до 60

**Комментарий:** Это "сердце" игры. Метод работает, пока is_running = True. Каждая итерация - это один кадр игры. Ограничение FPS нужно, чтобы игра работала с одинаковой скоростью на разных компьютерах.

## Класс Player

**Назначение:** Управление персонажем игрока.

### Метод `__init__(self, x, y)`

**Назначение:** Создание игрока в указанных координатах.

**Что делает:**
- Устанавливает начальную позицию
- Задает размер 50x50 пикселей
- Устанавливает зеленый цвет
- Задает скорость движения 5 пикселей за кадр

---

### Метод `move(self, keys)`

**Назначение:** Перемещение игрока по нажатым клавишам.

**Что делает:**
- Проверяет нажатые клавиши стрелок
- Двигает игрока в соответствующем направлении
- Не позволяет игроку выйти за границы экрана

**Комментарий:** Метод использует pygame.key.get_pressed() для получения состояния всех клавиш. Если игрок зажимает несколько стрелок, персонаж будет двигаться по диагонали.

---

### Метод `draw(self, screen)`

**Назначение:** Отрисовка игрока на экране.

**Что делает:** Рисует зеленый прямоугольник в текущих координатах игрока.

---

## Класс Enemy

**Назначение:** Управление врагами.

### Метод `__init__(self, x, y)`

**Назначение:** Создание врага в указанных координатах.

**Что делает:**
- Устанавливает позицию
- Задает размер 40x40 пикселей
- Устанавливает красный цвет
- Скорость 2 пикселя за кадр
- Начальное здоровье 2

---

### Метод `move_to_player(self, player_x, player_y)`

**Назначение:** Движение врага к игроку.

**Что делает:**
- Сравнивает координаты врага и игрока
- Если враг левее - двигается вправо
- Если враг правее - двигается влево
- Аналогично для вертикального движения

**Комментарий:** Это очень простой алгоритм движения. Враг всегда движется по прямой к игроку, без обхода препятствий. Этого достаточно для простой игры.

---

### Метод `hit(self)`

**Назначение:** Обработка попадания по врагу.

**Что делает:**
- Уменьшает здоровье врага на 1
- Возвращает True, если враг мертв (hp <= 0)

**Комментарий:** Возвращаемое значение используется в Game.check_collisions() для решения, удалять ли врага.

---

### Метод `draw(self, screen)`

**Назначение:** Отрисовка врага на экране.

**Что делает:** Рисует красный прямоугольник в текущих координатах врага.

---

## Класс Weapon

**Назначение:** Управление стрельбой и пулями.

### Метод `__init__(self, x, y)`

**Назначение:** Создание оружия в указанной позиции.

**Что делает:**
- Устанавливает позицию оружия (центр игрока)
- Создает пустой список для пуль
- Устанавливает задержку стрельбы в 0

---

### Метод `update_position(self, x, y)`

**Назначение:** Обновление позиции оружия (слежение за игроком).

**Что делает:** Устанавливает новые координаты оружия, обычно в центре игрока.

---

### Метод `shoot(self, target_x, target_y)`

**Назначение:** Создание новой пули.

**Что делает:**
- Проверяет, не на перезарядке ли оружие
- Создает новую пулю от текущей позиции к цели
- Добавляет пулю в список
- Устанавливает задержку в 10 кадров

**Комментарий:** Задержка (shoot_cooldown) нужна, чтобы игрок не мог стрелять слишком часто. Без нее за секунду можно было бы выпустить 60 пуль.

---

### Метод `update(self)`

**Назначение:** Обновление состояния оружия и пуль.

**Что делает:**
- Уменьшает таймер перезарядки
- Двигает все пули
- Удаляет пули, улетевшие за экран

**Комментарий:** Важно удалять улетевшие пули, иначе их накопится очень много и игра начнет тормозить.

---

### Метод `draw(self, screen)`

**Назначение:** Отрисовка всех пуль.

**Что делает:** Вызывает метод draw() для каждой пули в списке.

---

## Класс Bullet

**Назначение:** Управление отдельной пулей.

### Метод `__init__(self, x, y, target_x, target_y)`

**Назначение:** Создание пули с направлением к цели.

**Что делает:**
- Устанавливает начальную позицию (центр игрока)
- Вычисляет вектор от игрока к цели
- Нормализует вектор (делает длину 1)
- Умножает на скорость (10 пикселей/кадр)

**Комментарий:** Нормализация вектора - это математическая операция, которая позволяет пуле лететь точно в цель, а не только по 8 направлениям. Благодаря этому можно стрелять в любую точку экрана.

---

### Метод `move(self)`

**Назначение:** Движение пули.

**Что делает:**
- Добавляет к координатам вектор скорости
- Проверяет, не улетела ли пуля далеко за экран
- Возвращает True, если пуля должна исчезнуть

---

### Метод `draw(self, screen)`

**Назначение:** Отрисовка пули.

**Что делает:** Рисует желтый квадрат размером 8x8 пикселей.

## UI компоненты

### Класс Panel

**Назначение:** Базовая панель для UI элементов.

**Методы:**
- `add_button()` - добавление кнопки на панель
- `handle_events()` - передача событий кнопкам
- `draw()` - отрисовка панели и всех кнопок

---

### Класс Button

**Назначение:** Интерактивная кнопка.

**Методы:**
- `handle_event()` - обработка наведения и нажатия
- `draw()` - отрисовка кнопки с текстом

**Особенности:** Кнопка меняет цвет при наведении мыши.

---

### Класс MenuPanel

**Назначение:** Главное меню игры.

**Особенности:** Наследуется от Panel и автоматически создает кнопку START по центру экрана.

---

## Принципы ООП, использованные в проекте

1. **Инкапсуляция** - каждый класс отвечает только за свою функциональность
2. **Наследование** - MenuPanel наследуется от Panel
3. **Полиморфизм** - все игровые объекты имеют методы draw() и update()
4. **Композиция** - Game содержит Player, Weapon, Enemy
